# 总览

Memorai 的架构由一个原则塑造：**原始时间轴即真相；其上的所有内容都是可被替换的解释。**每一层都有不同的生命周期,各层之间通过清晰的接口通信,从而独立演进。

## 四个层级

```
┌──────────────────────────────────────────────────────────────────┐
│  Tier 3 — 索引  （计算生成,可丢弃）                              │
│                                                                  │
│  BM25 · Vector · Tag · Time · Participant · Topic                │
│  自动从 Tier 1 + 2 重建。                                        │
└──────────────────────────────────────────────────────────────────┘
                              ▲
┌──────────────────────────────────────────────────────────────────┐
│  Tier 2.5 — MemoryEvents  （生命周期受管理）                     │
│                                                                  │
│  state / transition / happening 事件,由 EventIdentifier          │
│  识别。状态事件可被取代。                                        │
└──────────────────────────────────────────────────────────────────┘
                              ▲
┌──────────────────────────────────────────────────────────────────┐
│  Tier 2 — 注解  （可重生）                                       │
│                                                                  │
│  summary · facts · tags · embedding · knowledge triples          │
│  当更好的抽取器上线时,由 reAnnotate() 替换。                     │
└──────────────────────────────────────────────────────────────────┘
                              ▲
┌──────────────────────────────────────────────────────────────────┐
│  Tier 1 — 原始时间轴  （不可变,仅追加）                          │
│                                                                  │
│  事件发生时的精确内容。永不改写。                                │
└──────────────────────────────────────────────────────────────────┘
```

## 各组成部分

| 概念 | 所属层 | 描述 |
|---|---|---|
| [`Event`](/zh/api/memorai#recordevent) | 输入 | 传递给 `recordEvent()` 的形状。Text / image / audio / video / file / observation / custom。 |
| [`MemoryNode`](/zh/concepts/memory-nodes) | Tier 1 + 2 | 已持久化的记忆。同时承载 `raw`(Tier 1)和 `annotations`(Tier 2)。具有 `level`：`segment` / `atomic_action` / `episode`。 |
| [`MemoryEvent`](/zh/concepts/memory-events) | Tier 2.5 | 从原始节点中识别出的语义事件。三种类型：`state`、`transition`、`happening`。携带自身的嵌入和取代链。 |
| [分层记忆演进](/zh/concepts/evolution) | Tier 1 | 后台进程,将原始片段聚类为原子动作和情节,以支持局部性感知的召回。 |
| [事件识别](/zh/concepts/memory-events) | Tier 2.5 | 每次 `evolve()` 步骤中,将原始节点转换为 MemoryEvents。默认运行 LLM,但接口接受任何自定义识别器。 |
| [多路径召回](/zh/concepts/retrieval) | 读取 | 语义 + BM25 + 标签 + 时间 + 身份(+ 事件级路径)扇出后通过 RRF 融合。 |
| [跨代理画像](/zh/concepts/cross-agent) | 读写 | 每代理的 `writePolicy` / `readPolicy`,使多个代理可共享一个存储,并以不同粒度查看。 |

## 三条不变式

1. **Tier 1 仅追加。**一旦写入,原始事件不可变。任何抽取器、任何演进步骤、任何升级都不会改写它们。这是*永不忘记*的承诺——即便上层的一切都变化,实际观察到的内容始终保持完整。

2. **Tier 2 + Tier 2.5 可重生。**注解和 MemoryEvents 是对原始记录的*解释*。当更好的 LLM、更聪明的提示词或领域专用的识别器到来时,你可以在全部历史上重新运行它们：
   - [`reAnnotate()`](/zh/api/memorai#reannotate) —— 从 Tier 1 刷新 Tier 2
   - [`identifyRecent()`](/zh/api/memorai#identifyrecent) —— 从 Tier 1 刷新/扩展 Tier 2.5
   原始存储是持久的基底;其上一切都可重建。

3. **Tier 3 可丢弃。**索引是计算生成的产物。可随时丢弃并重建。存储适配器会自动维护它们。

## 独特能力

像 mem0 和 Zep 这样的顶级记忆库在摄入时就把记忆压缩成单层派生层。当它们的抽取质量提升时,旧记忆并不受益。当一个状态断言过时,新旧都保存在存储中,代理只能在运行时自行判断哪个是当前的。

Memorai 的三层模型让你能够：

- **升级模型,免费重建索引。**新 LLM 上线 → 调用 `reAnnotate({ extractor: newLLM })` → 全部历史在不丢失源数据的情况下重新汇总。
- **同时持有多种解释。**在同一 Tier 1 上运行两个识别器(一个通用、一个领域专用),并从任一处召回。
- **追溯源头。**每个 MemoryEvent 都指向识别它的原始节点。每个被召回的记忆都携带使其浮现的路径。
- **回放历史。**通过 `validAt` 过滤召回,查看代理在某一特定时间点的信念。
